# EPDMS加权指标硬约束边界详解

## 概述

在EPDMS（扩展预测驾驶员模型分数）评估体系中，加权指标反映了驾驶质量和效率的多个方面。与乘数指标不同，加权指标的违规不会导致整个EPDMS分数为0，但每个指标本身有严格的硬约束边界，违反这些边界将导致该指标得分为0。

本文档详细解释每个加权指标的硬约束边界，说明何时该指标分数为0。

## 1. 碰撞时间 (Time To Collision, TTC)

### 硬约束边界
**分数为0的条件**：在评估时间窗口（默认1秒）内存在碰撞风险，且自车应承担责任。

### 具体触发条件
1. **预测碰撞**：预测未来1秒内自车位置与环境中对象发生碰撞
2. **非静止对象**：碰撞对象不是静止车辆（速度 > 0.005 m/s）
3. **责任判断**：必须同时满足以下条件：
   - 前车在自车前方（`is_agent_ahead()`返回True）
   - 自车在多个车道、交叉口或非可行驶区域

### 关键逻辑
- 使用外推多边形表示自车在未来时刻的形状
- 排除已碰撞对象和静止车辆
- 考虑自车位置对责任的影响

### 代码实现
- 代码位置：`pdm_scorer.py`第492-580行
- 关键代码行：`ttc_scores[proposal_idx] = np.minimum(ttc_scores[proposal_idx], 0.0)`（第575行）

## 2. 自我进度 (Ego Progress, EP)

### 特殊说明：无硬约束0分边界
**重要区别**：EP是连续值指标[0,1]，**没有硬约束0分边界**。

### 分数计算规则
- **归一化计算**：实际前进距离 / 最大前进距离
- **0分条件**：实际前进距离 ≤ 0（后退或原地不动）
- **但这不是"违规"**：只是性能差，不是安全违规

### 参数说明
- `progress_distance_threshold`: 5.0米（归一化阈值）
- 如果最大距离 ≤ 5.0米，所有轨迹得分为1

## 3. 车道保持 (Lane Keeping, LK)

### 硬约束边界
**分数为0的条件**：在连续2秒内横向偏差持续超过0.5米（在非交叉口区域）。

### 具体触发条件
1. **偏差计算**：自车中心点与车道中心线横向偏差 > 0.5米
2. **连续时间**：偏差持续至少2秒（连续20个时间步，间隔0.1秒）
3. **区域限制**：仅在非交叉口区域评估
4. **交叉口处理**：在交叉口区域不评估此指标

### 关键逻辑
- 使用`ego_position.distance(centerline)`计算横向偏差
- 检测连续超过阈值的时间
- 使用`self._drivable_area_map.is_in_layer()`检测交叉口

### 参数
- `lane_keeping_deviation_limit`: 0.5米（横向偏差阈值）
- `lane_keeping_horizon_window`: 2.0秒（连续违规时间窗口）

### 代码实现
- 代码位置：`pdm_scorer.py`第615-654行
- 关键代码：`if consecutive_exceeds >= continuous_steps_required: lane_keeping_scores[proposal_idx] = 0.0`（第650-652行）

## 4. 历史舒适度 (History Comfort, HC)

### 硬约束边界
**分数为0的条件**：任意一个舒适度维度超出阈值范围。

### 六个硬约束维度
1. **纵向加速度**：超出[-4.05, 2.40] m/s²
2. **横向加速度**：超出[-4.89, 4.89] m/s²
3. **总加加速度**：超出[-8.37, 8.37] m/s³
4. **纵向加加速度**：超出[-4.13, 4.13] m/s³
5. **横摆角加速度**：超出[-1.93, 1.93] rad/s²
6. **横摆角速度**：超出[-0.95, 0.95] rad/s

### 评估逻辑
- **所有维度必须同时满足**：任一违规即得0分
- **轨迹拼接**：结合过去的人类驾驶轨迹和当前预测轨迹
- **平滑处理**：使用Savitzky-Golay滤波器平滑加速度数据
- **评估函数**：`ego_is_comfortable()`

### 关键特点
- 考虑历史状态对当前舒适度的影响
- 使用扩展状态数组进行评估
- 所有时间步都必须舒适

## 5. 扩展舒适度 (Extended Comfort, EC)

### 硬约束边界
**分数为0的条件**：任意一个特征的RMS差异超出阈值。

### 四个硬约束维度
1. **加速度差异**：RMS > 0.7 m/s²
2. **加加速度差异**：RMS > 0.5 m/s³
3. **横摆角速度差异**：RMS > 0.1 rad/s
4. **横摆角加速度差异**：RMS > 0.1 rad/s²

### 评估逻辑
1. **特征提取**：比较连续两帧的轨迹特征差异
2. **RMS计算**：计算每个特征的RMS（均方根）差异
3. **阈值比较**：所有RMS值必须同时满足阈值
4. **评估函数**：`ego_is_two_frame_extended_comfort()`

### 关键函数
- `extract_features()`：提取加速度、加加速度、横摆角速度、横摆角加速度
- `calculate_rms()`：计算RMS差异
- 比较连续两帧轨迹的特征一致性

### 参数
- `acceleration_threshold`: 0.7 m/s²
- `jerk_threshold`: 0.5 m/s³
- `yaw_rate_threshold`: 0.1 rad/s
- `yaw_accel_threshold`: 0.1 rad/s²

### 代码实现
- 代码位置：`pdm_comfort_metrics.py`第430-467行

## 硬约束与软约束对比

| 指标 | 硬约束0分条件 | 软约束（性能差） | 影响范围 |
|------|--------------|------------------|----------|
| TTC | 存在责任碰撞风险 | 无 | 只影响TTC分数 |
| EP | 无（只有性能0分） | 前进距离为0或负 | 只影响EP分数 |
| LK | 连续2秒偏差>0.5m | 临时偏差 | 只影响LK分数 |
| HC | 任一舒适度维度超标 | 接近阈值 | 只影响HC分数 |
| EC | 任一RMS差异超标 | 差异较大但未超标 | 只影响EC分数 |

## 关键区别

### 1. 乘数指标 vs 加权指标硬约束
- **乘数指标**：违反即整个EPDMS为0（或减半）
- **加权指标硬约束**：只影响该指标分数为0，不影响其他指标

### 2. 实际影响差异
- **加权指标的硬约束0分**：只会降低加权平均部分
- **乘数指标的违规**：使整个乘积为0

### 3. 设计哲学
- **安全违规（乘数指标）**：惩罚更重，影响整个分数
- **性能/舒适度违规（加权指标）**：惩罚较轻，只影响部分分数

## 调试建议

### 1. TTC分数为0
- 检查是否存在碰撞风险预测
- 分析自车位置（是否在多个车道或交叉口）
- 检查相对位置判断逻辑

### 2. LK分数为0
- 测量横向偏差是否持续超过0.5米
- 检查是否在非交叉口区域
- 确认连续违规时间是否达到2秒

### 3. HC分数为0
- 检查六个舒适度维度的具体数值
- 分析哪个维度超出阈值
- 考虑历史轨迹的影响

### 4. EC分数为0
- 计算连续两帧的特征差异
- 检查RMS值是否超出阈值
- 分析轨迹的平滑性

## 总结

EPDMS加权指标的硬约束边界体现了对驾驶质量和舒适度的严格要求，但设计上避免了过度惩罚。这些边界：

1. **反映实际驾驶标准**：基于人类驾驶数据和工程经验
2. **区分严重程度**：安全违规（乘数指标）比性能违规惩罚更重
3. **提供诊断信息**：每个指标的0分都有明确的触发条件
4. **支持系统优化**：帮助开发者识别和解决具体问题

理解这些硬约束边界对于优化自动驾驶系统性能、诊断问题原因以及提高EPDMS分数至关重要。
