# EPDMS (扩展预测驾驶员模型分数) 技术文档

## 1. 概述

EPDMS (Extended Predictive Driver Model Score，扩展预测驾驶员模型分数) 是NAVSIM v2中引入的自动驾驶评估指标，用于全面评估自动驾驶系统的性能。该指标在NAVSIM v1的PDMS基础上进行了扩展，增加了更多维度的评估指标和更精细的惩罚机制，旨在提供更全面、更公平的自动驾驶系统评估。

### 设计目标

1. **全面性**：涵盖安全性、合规性、舒适性和效率等多个维度
2. **公平性**：通过虚假正例惩罚过滤机制，避免对必要违规行为的不公平惩罚
3. **效率性**：通过两阶段伪闭环评估，在保持评估准确性的同时大幅降低计算成本
4. **可解释性**：提供详细的子指标分数，便于问题诊断和系统优化

## 2. 指标组成

EPDMS由两类指标组成：**乘数指标**和**加权指标**。

### 乘数指标 (Multiplicative Metrics)

乘数指标为二进制或三值指标，违反则整个分数为零或减半。这些指标主要评估安全性关键行为：

| 指标 | 简称 | 取值范围 | 权重 | 描述 |
|------|------|----------|------|------|
| 无过错碰撞 | NC | {0, 0.5, 1} | 乘数 | 评估是否有过错碰撞发生 |
| 可行驶区域合规性 | DAC | {0, 1} | 乘数 | 评估是否在可行驶区域内行驶 |
| 行驶方向合规性 | DDC | {0, 0.5, 1} | 乘数 | 评估是否违反行驶方向规则 |
| 交通信号灯合规性 | TLC | {0, 1} | 乘数 | 评估是否遵守交通信号灯 |

### 加权指标 (Weighted Metrics)

加权指标为连续值指标，通过加权平均组合。这些指标评估驾驶质量和效率：

| 指标 | 简称 | 取值范围 | 权重 | 描述 |
|------|------|----------|------|------|
| 自我进度 | EP | [0, 1] | 5 | 沿中心线的前进距离 |
| 碰撞时间 | TTC | {0, 1} | 5 | 评估碰撞风险时间 |
| 车道保持 | LK | {0, 1} | 2 | 评估车道保持能力 |
| 历史舒适度 | HC | {0, 1} | 2 | 结合历史状态的舒适性评估 |
| 扩展舒适度 | EC | {0, 1} | 2 | 评估连续帧间的一致性 |

## 3. 详细技术规格

### 3.1 乘数指标计算详解

乘数指标通过乘积方式影响最终分数，任何违规都会导致分数显著降低。每个指标的具体计算方法如下：

#### 3.1.1 无过错碰撞 (No At-Fault Collisions, NC)

**取值范围**: {0, 0.5, 1}
- 1: 无碰撞
- 0.5: 与非智能体对象碰撞
- 0: 与智能体对象碰撞

**计算方法（代码位置：`pdm_scorer.py`第363-420行）**:
1. **碰撞检测**: 遍历每个时间步，检测自车边界框与环境中其他对象的碰撞
   - 使用空间查询 `self._observation[time_idx].query(ego_polygons, predicate="intersects")` 检测碰撞
   - 排除已经碰撞过的对象和红灯对象
2. **责任判断**: 根据碰撞类型和自车位置判断责任
   - 前部碰撞 (`ACTIVE_FRONT_COLLISION`) 或静止车辆碰撞 (`STOPPED_TRACK_COLLISION`): 智能体过错
   - 侧面碰撞 (`ACTIVE_LATERAL_COLLISION`): 仅在自车在多个车道或非可行驶区域时才判定为过错
   - 否则视为无过错碰撞，将对象加入已碰撞列表
3. **分数计算**:
   - 与智能体对象碰撞: 0分
   - 与非智能体对象碰撞: 0.5分
   - 无碰撞: 1分
   - 取所有碰撞中的最低分数

**关键逻辑**:
- 使用 `get_collision_type()` 函数分类碰撞类型
- 通过 `ego_in_multiple_lanes_or_nondrivable_area` 判断自车位置
- 记录第一次碰撞的时间索引 (`_collision_time_idcs`)

#### 3.1.2 可行驶区域合规性 (Drivable Area Compliance, DAC)

**取值范围**: {0, 1}
- 1: 始终在可行驶区域内
- 0: 至少有一个时刻在非可行驶区域

**计算方法（代码位置：`pdm_scorer.py`第422-429行）**:
1. **区域检查**: 检查自车四个角点是否在可行驶区域多边形内
   - 可行驶区域包括：道路区块 (`ROADBLOCK`)、交叉口 (`INTERSECTION`)、可行驶区域 (`DRIVABLE_AREA`)、停车场 (`CARPARK_AREA`)
2. **违规判定**: 只要有一个角点不在任何可行驶多边形内，即判为违规
3. **分数计算**:
   - 所有时刻都在可行驶区域内: 1分
   - 至少有一个时刻在非可行驶区域: 0分

**关键逻辑**:
- 使用 `self._ego_areas[:, :, EgoAreaIndex.NON_DRIVABLE_AREA].any(axis=-1)` 检测违规
- 基于预计算的可行驶区域地图 (`PDMDrivableMap`) 进行空间查询

#### 3.1.3 行驶方向合规性 (Driving Direction Compliance, DDC)

**取值范围**: {0, 0.5, 1}
- 1: 完全合规
- 0.5: 轻微违规
- 0: 严重违规

**计算方法（代码位置：`pdm_scorer.py`第431-474行）**:
1. **逆行检测**: 计算自车在逆行车道上的行驶距离
   - 通过 `self._ego_areas[:, :, EgoAreaIndex.ONCOMING_TRAFFIC]` 识别逆行时刻
   - 排除交叉口区域的逆行（在交叉口允许短暂逆行）
2. **距离累积**: 在滑动时间窗口（默认1秒）内累积逆行距离
3. **阈值判断**:
   - 累计距离 < 2.0米: 合规 (1.0)
   - 2.0米 ≤ 累计距离 < 6.0米: 轻微违规 (0.5)
   - 累计距离 ≥ 6.0米: 严重违规 (0.0)

**相关参数**:
- `driving_direction_horizon`: 1.0秒（评估时间窗口）
- `driving_direction_compliance_threshold`: 2.0米（合规阈值）
- `driving_direction_violation_threshold`: 6.0米（严重违规阈值）

#### 3.1.4 交通信号灯合规性 (Traffic Light Compliance, TLC)

**取值范围**: {0, 1}
- 1: 遵守所有交通信号灯
- 0: 违反至少一个交通信号灯

**计算方法（代码位置：`pdm_scorer.py`第582-613行）**:
1. **红灯检测**: 遍历每个时间步，检测自车边界框与红灯区域的交集
   - 使用 `self._observation[time_idx].query(ego_polygons, predicate="intersects")` 检测相交
2. **违规判定**: 检查相交对象的token是否以红灯标识 (`red_light_token`) 开头
   - 只要有一个时刻与红灯区域相交，即判为违规
3. **分数计算**:
   - 遵守所有交通信号灯: 1分
   - 违反至少一个交通信号灯: 0分

**关键逻辑**:
- 使用特殊的token标识系统识别红灯对象
- 只要有一次违规，分数即为0，后续检测可提前终止

### 3.2 加权指标计算详解

加权指标通过加权平均组合，反映驾驶质量和效率的多个方面。每个指标的具体计算方法如下：

#### 3.2.1 自我进度 (Ego Progress, EP)

**取值范围**: [0, 1]（归一化值）

**计算方法（代码位置：`pdm_scorer.py`第476-490行）**:
1. **距离计算**: 沿中心线计算实际前进距离
   - 使用 `self._centerline.project()` 计算起点和终点在中心线上的投影
   - 前进距离 = 终点投影 - 起点投影
2. **最大距离**: 在所有候选轨迹中找出最大前进距离
3. **归一化**:
   - 如果最大距离 > 5.0米: 归一化分数 = 实际距离 / 最大距离
   - 如果最大距离 ≤ 5.0米: 所有轨迹得分为1
4. **负距离处理**: 如果前进距离为负（后退），则取0

**相关参数**:
- `progress_distance_threshold`: 5.0米（归一化阈值）

#### 3.2.2 碰撞时间 (Time To Collision, TTC)

**取值范围**: {0, 1}
- 1: 在评估时间内无碰撞风险
- 0: 存在碰撞风险

**计算方法（代码位置：`pdm_scorer.py`第492-580行）**:
1. **未来预测**: 预测未来1秒内自车的位置（考虑当前速度）
   - 创建外推多边形 (`coords_exterior_time_steps`) 表示自车在未来时刻的形状
2. **碰撞检测**: 检测预测多边形与环境中对象的碰撞
   - 排除静止车辆（速度 < 0.005 m/s）
   - 排除已碰撞对象
3. **责任判断**: 根据相对位置和方向判断责任
   - 前车在自车前方且自车在多个车道或交叉口: 自车责任
   - 否则: 无责任
4. **分数计算**:
   - 在评估时间内无碰撞风险: 1分
   - 存在碰撞风险: 0分
   - 记录第一次违规的时间索引 (`_ttc_time_idcs`)

**关键逻辑**:
- 使用 `is_agent_ahead()` 和 `is_agent_behind()` 判断相对位置
- 考虑自车位置（多个车道、交叉口）对责任的影响

#### 3.2.3 车道保持 (Lane Keeping, LK)

**取值范围**: {0, 1}
- 1: 保持良好的车道居中
- 0: 车道保持不良

**计算方法（代码位置：`pdm_scorer.py`第615-654行）**:
1. **偏差计算**: 计算自车中心点与车道中心线的横向偏差
   - 使用 `ego_position.distance(centerline)` 计算距离
2. **连续违规检测**: 检测连续超过阈值的时间
   - 如果在连续2秒内横向偏差持续超过0.5米，则判为不合格
3. **交叉口处理**: 在交叉口区域不评估此指标
   - 使用 `self._drivable_area_map.is_in_layer()` 检测交叉口
4. **分数计算**:
   - 保持良好的车道居中: 1分
   - 车道保持不良: 0分

**相关参数**:
- `lane_keeping_deviation_limit`: 0.5米（横向偏差阈值）
- `lane_keeping_horizon_window`: 2.0秒（连续违规时间窗口）

#### 3.2.4 历史舒适度 (History Comfort, HC)

**取值范围**: {0, 1}
- 1: 所有时刻都舒适
- 0: 至少有一个时刻不舒适

**计算方法（代码位置：`pdm_scorer.py`第656-700行）**:
1. **轨迹拼接**: 结合过去的人类驾驶轨迹和当前预测轨迹
   - 使用 `self._human_past_trajectory` 获取历史轨迹
   - 将历史状态和预测状态拼接成扩展状态数组
2. **舒适度评估**: 评估多个舒适度维度：
   - 纵向加速度：[-4.05, 2.40] m/s²
   - 横向加速度：[-4.89, 4.89] m/s²
   - 总加加速度：[-8.37, 8.37] m/s³
   - 纵向加加速度：[-4.13, 4.13] m/s³
   - 横摆角加速度：[-1.93, 1.93] rad/s²
   - 横摆角速度：[-0.95, 0.95] rad/s
3. **平滑处理**: 使用Savitzky-Golay滤波器平滑加速度数据
4. **阈值判断**: 所有维度都在阈值内才判为舒适

**关键逻辑**:
- 使用 `ego_is_comfortable()` 函数进行评估
- 考虑历史状态对当前舒适度的影响

#### 3.2.5 扩展舒适度 (Extended Comfort, EC)

**取值范围**: {0, 1}
- 1: 连续帧间变化平滑
- 0: 连续帧间变化剧烈

**计算方法（代码位置：`pdm_comfort_metrics.py`第430-467行）**:
1. **特征提取**: 比较连续两帧的轨迹特征差异
   - 提取加速度、加加速度、横摆角速度、横摆角加速度四个特征
2. **RMS计算**: 计算每个特征的RMS（均方根）差异
   - 使用 `calculate_rms()` 函数计算连续帧间的特征差异
3. **阈值比较**:
   - 加速度差异：≤ 0.7 m/s²
   - 加加速度差异：≤ 0.5 m/s³
   - 横摆角速度差异：≤ 0.1 rad/s
   - 横摆角加速度差异：≤ 0.1 rad/s²
4. **分数计算**: 所有特征的RMS差异都在阈值内才判为合格

**相关参数**:
- `acceleration_threshold`: 0.7 m/s²
- `jerk_threshold`: 0.5 m/s³
- `yaw_rate_threshold`: 0.1 rad/s
- `yaw_accel_threshold`: 0.1 rad/s²

## 4. EPDMS计算公式

EPDMS的计算公式如下：

$$
\text{EPDMS} = \left(\prod_{m\in\{NC, DAC, DDC, TLC\}} \text{filter}_m(\text{agent}, \text{human})\right) \cdot  \left( \frac{\sum_{m \in \{TTC, EP, HC, LK, EC\}} w_m \cdot \text{filter}_m(\text{agent}, \text{human}) }{\sum_{m\in \{TTC, EP, HC, LK, EC\}} w_m}\right)
$$

其中过滤函数定义为：

$$
\text{filter}_m(\text{agent}, \text{human}) = \begin{cases}
1.0 & \text{if } m(\text{human}) = 0 \\
m(\text{agent}) & \text{otherwise.}
\end{cases}
$$

### 虚假正例惩罚过滤

过滤函数实现了一种重要的公平性机制：**虚假正例惩罚过滤**。当人类驾驶员在相同情况下也会违规时，不惩罚自动驾驶系统。这避免了在以下情况下不公平的惩罚：

1. 必须短暂进入对向车道以绕过静止障碍物
2. 在复杂交通情况下必须做出的必要违规
3. 由于环境限制无法完全遵守规则的情况

## 5. 两阶段伪闭环评估

NAVSIM v2引入两阶段伪闭环评估，在保持开环效率的同时模拟闭环行为。

### 第一阶段：初始场景评估
- 使用原始场景进行4秒模拟
- 背景交通参与者按照记录的未来轨迹运动
- 自车基于LQR控制器按照规划轨迹运动
- 计算初始EPDMS分数

### 第二阶段：后续场景评估
- 使用预计算的多个后续场景进行评估
- 后续场景通过从初始场景出发、应用不同4秒规划进行模拟生成
- 每个后续场景从不同的状态开始（例如横向偏移、不同速度）

### 加权聚合
1. 根据第一阶段的实际结束位置为后续场景分配权重
2. 使用高斯核函数：距离越近权重越高
3. 计算第二阶段分数的加权平均
4. 最终分数 = 第一阶段分数 × 第二阶段加权分数

这种方法的优势：
- **计算效率**：相比传统闭环仿真减少6倍计算量
- **行为捕捉**：能够捕捉错误恢复和行为偏离等闭环特性
- **可并行性**：所有评估可以并行计算

## 6. 实现架构

### 核心模块

#### 1. PDMScorer类 (`pdm_scorer.py`)
- 主评分器类，实现所有指标的计算
- 包含配置类`PDMScorerConfig`，定义权重和阈值
- 提供`score_proposals()`方法进行批量评分

#### 2. 舒适度指标 (`pdm_comfort_metrics.py`)
- 实现舒适度相关的特征提取和评估
- 包含加速度、加加速度、横摆角等特征的计算
- 提供`ego_is_comfortable()`和`ego_is_two_frame_extended_comfort()`方法

#### 3. 枚举定义 (`pdm_enums.py`)
- 定义状态索引、指标索引等枚举
- 包括`MultiMetricIndex`和`WeightedMetricIndex`

#### 4. 主评分函数 (`pdm_score.py`)
- 提供`pdm_score()`和`pdm_score_from_interpolated_trajectory()`函数
- 处理轨迹转换和评分流程
- 实现虚假正例惩罚过滤逻辑

### 数据流

1. **输入准备**
   - 从MetricCache加载场景数据
   - 转换预测轨迹为全局坐标系
   - 准备仿真所需的所有输入

2. **仿真执行**
   - 使用PDMSimulator进行轨迹仿真
   - 应用交通参与者策略
   - 生成仿真后的状态序列

3. **指标计算**
   - 计算所有乘数指标
   - 计算所有加权指标
   - 应用虚假正例过滤

4. **分数聚合**
   - 计算乘数指标乘积
   - 计算加权指标平均值
   - 应用两阶段聚合（如果启用）

## 8. 与PDMS的对比

### NAVSIM v1 PDMS
- **乘数指标**: NC, DAC
- **加权指标**: TTC, EP, C（舒适度）
- **权重**: EP:5, TTC:5, C:2
- **评估方式**: 单阶段4秒模拟
- **无虚假正例过滤**

### NAVSIM v2 EPDMS
- **新增乘数指标**: DDC, TLC
- **新增加权指标**: LK, EC
- **改进指标**: HC（增强的历史舒适度）
- **新增功能**: 虚假正例惩罚过滤
- **评估方式**: 两阶段伪闭环评估
- **权重调整**: 更全面的权重分配

### 主要改进
1. **更全面的安全评估**: 增加了行驶方向和交通信号灯合规性
2. **更精细的驾驶质量评估**: 增加了车道保持和扩展舒适度
3. **更公平的评估**: 引入虚假正例惩罚过滤
4. **更接近闭环评估**: 两阶段伪闭环聚合

## 10. 总结

EPDMS代表了自动驾驶评估领域的重要进步，它通过：

1. **多维度的评估体系**：涵盖安全、合规、舒适、效率等多个方面
2. **公平的评估机制**：避免对必要违规的不公平惩罚
3. **高效的评估方法**：在保持评估质量的同时大幅降低计算成本
4. **详细的诊断信息**：提供丰富的子指标分数，便于系统优化

EPDMS已成功应用于多个自动驾驶挑战赛，包括AGC2025 NAVSIM端到端驾驶挑战赛，证明了其在自动驾驶系统评估中的有效性和实用性。
